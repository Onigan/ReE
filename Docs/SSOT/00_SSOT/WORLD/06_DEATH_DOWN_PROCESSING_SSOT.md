# 06_DEATH_DOWN_PROCESSING_SSOT.md（SSOT v1.1）

更新: 2026-01-24（v1.1）
出典: 本会話（あなたの方針）
重要: HP0は即死ではない。死亡判定と戦闘不能判定は別。

---

## 1. 状態の考え方（確定）
- 戦闘不能（Down）：行動不能・意識喪失に近いが、即死ではない
- 死亡（Dead）：転生へ進む（PCは例外なく転生が基本）
- クリティカル等で即死演出（首を落とされる等）は「即死が確定した場合のみ」発生しうる

---

## 2. Down→Dead への移行条件（数値なし運用 / 確定）
Downのまま放置されると、敵や環境により処置（Processing）が進行し Dead に至ることがある。

例：
- 放置される
- 追撃される
- 運搬される
- “処置開始”が宣言される（Processing P1開始）

---

## 3. Processing（P0〜P3）テーブル（確定：枠組み）
- P0：未処置（戦闘後すぐ/戦場放置）
- P1：処置開始（運搬、拘束、捕食、同化、素材化の開始など）
- P2：進行（身体・意識・所属が変化し始める段階。救出/蘇生の難易度が上がる）
- P3：完了（実質Dead相当。転生/喪失/不可逆に近いが、ルール上は“蘇生手段”で例外を作れる）

※“不可逆の永久汚染”は禁止方針なので、P3後でも救済ルートは「超希少」枠で残す。

---

## 4. 暗転・断片演出（確定：面白さの核）
- Down時点で暗転し、他者の治療/捕虜化/処置などが確定した段階で目覚める（テンポ維持）
- 進行に応じて「ただの暗転」ではなく、世界が見える断片演出を挿入できる
- 同化/随伴化が起きる場合の断片例：
  - 味方に襲いかかる片鱗
  - 安心感と意思の希薄化
  - “救いを周囲へ与えたい”衝動
